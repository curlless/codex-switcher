# Code of Conduct

This repository is maintained as a practical engineering workspace. Keep discussion
technical, respectful, and useful.

## Expected behavior

- discuss code and behavior, not people
- use concrete examples when reporting bugs or regressions
- assume good intent, but challenge weak technical reasoning directly
- keep review feedback specific and actionable
- protect credentials, customer data, and private infrastructure details

## Unacceptable behavior

- harassment, insults, or personal attacks
- publishing credentials, private tokens, or private user data
- spam, trolling, or intentionally disruptive review behavior
- repeated low-signal noise after maintainers ask to stop

## Scope

This applies to issues, pull requests, discussions, and any repository-linked
communication channel.

## Enforcement

Maintainers may edit, hide, lock, or remove content that hurts collaboration or
security. Repeated abuse can result in a permanent ban from the repository.

## Reporting

Open a private security report for sensitive issues. For conduct problems that are
not security-related, contact the maintainers through repository discussions or a
private channel you already use with the team.
